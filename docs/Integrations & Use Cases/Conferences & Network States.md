Conferences generate extraordinary value — hallway conversations, workshop breakthroughs, panel debates — and almost all of it vanishes. Attendees leave with business cards and fragmented notes. The collective intelligence that emerged over three days dissipates within a week.

Bonfires captures and preserves the sensemaking that happens at your event.

## What It Does

**Real-time knowledge capture.** Agents deployed across event channels — main groups, workshop discussions, topic threads — extract entities, relationships, and insights into a growing knowledge graph throughout the event.

**Queryable event intelligence.** Attendees can ask: "Who else is working on X?" "What were the main tensions in that panel?" "What's happening today?" Answers are synthesized from all conversations across the event.

**Persistent artifact.** The knowledge graph survives the event. It becomes a shareable proof of what happened — useful for attendees who missed sessions, organisers writing reports, sponsors evaluating impact, and future editions building on accumulated context.

**Multi-agent deployment.** Different venues, tracks, or communities each get their own agent, all feeding into a shared Bonfire. Cross-venue patterns become visible that no single track could have seen.

## Deployments

### Edge City Patagonia — Edgegardo

**4 weeks · 300+ participants · Network state pop-up village**

The agent listened to the main Telegram group and side chats, building a knowledge graph in real-time. Residents queried what was happening across the village, explored connections through the graph, and used [[HyperBlogs (Technical)|DataRooms]] for targeted research. Institutional memory persisted after the event ended.

### Devconnect Buenos Aires — Multi-Agent Bonfire

**1 week · Multiple venues · Hundreds of participants**

Four agents — World of DeSci Hub, Devconnect General, Regen Hub, Wen Tokin (BluntDAO) — each maintained local context while feeding a shared knowledge graph. Cross-venue patterns emerged that no single venue could have tracked.

### Synkora — Mapping Synchronicities

**Ongoing research · Edge City consciousness residency**

Researcher Tugce Tasci used a Bonfire to track synchronicities as data points, using the knowledge graph to identify patterns across participants. The AI surfaced probability attractors, symbolic clusters, and validation feedback loops that would have been impossible with traditional methods.

## For Event Organisers

### What We Need From You

- **Digital access** — Add the agent to event Telegram groups or Discord
- **Context** — Event programme, speaker list, key themes
- **Endorsement** — Introduce the agent to attendees; facilitators prompting usage is the difference between a passive tool and an active layer of the event experience
- **Timeline** — 1–2 weeks before the event for setup and context seeding

### What It Costs

Compute scales with participants and duration. A 100-person event over 4 weeks runs roughly $60–240 in inference costs. Full pricing through [[docs/General Info/Team|Bonfires Labs]].
