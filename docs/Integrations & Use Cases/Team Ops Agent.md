Your team's knowledge is scattered. Decisions live in Telegram threads, meeting context sits in Fireflies, tasks are tracked in ClickUp, and the connections between them exist only in people's heads.

A Team Ops Agent unifies this into a single, queryable knowledge layer.

## What It Does

**Captures team discussions.** The agent sits in your team's Telegram group(s), processing conversations into a knowledge graph. Decisions, action items, context, and relationships are extracted automatically.

**Integrates with your tools.** Through MCP tool connections, the agent can interface with:

- **Fireflies** — Meeting transcripts flow into the knowledge graph
- **ClickUp** — Task context enriches the agent's understanding of who's working on what
- **Google Drive** — Documents become searchable knowledge
- **Custom integrations** — Any MCP-compatible tool can be added

**Answers operational questions.** "What did we decide in Monday's standup?" "Who's handling the partnership outreach?" "What's the context behind this ClickUp ticket?" The agent searches across all captured knowledge — conversations, meetings, and documents.

**Reduces status meetings.** When the agent already tracks ongoing work through episodic memory, meetings can focus on deciding and unblocking rather than status reporting.

## What This Looks Like

1. **Agent joins your team Telegram** — main group, topic-specific chats, or both
2. **Configure tool integrations** — connect Fireflies, ClickUp, or other tools via MCP
3. **Team talks normally** — no behaviour change required
4. **Knowledge builds automatically** — every 20 minutes, conversations and meeting transcripts are processed
5. **Query anytime** — tag the agent for instant context from your team's full operational history

## Why Telegram

Your team already lives there. No new app to adopt, no dashboard to check. The agent is alongside your team, in the group chat, with typing indicators and natural conversation.

For teams that prefer Discord, the same agent architecture works across channels and servers.
